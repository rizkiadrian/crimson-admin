Milestone.event_type maps to UserEvent types:
  • "user_registered" → Milestone 1
  • "email_verified" → Milestone 2 (Client)
  • "mitra_approved" → Milestone 2 (Mitra) *new event*
  • "first_deposit" → Milestone 3 (Client)
  • "first_transaction" → Milestone 3 (Mitra)

⚡ Key Design Decisions

Strengths

  • Fully normalized — no JSON blobs for critical data
  • Tier bonus calculated at disburse time (not stored on tier change)
  • Referral codes per-campaign — user bisa punya kode berbeda per campaign
  • Soft delete on campaigns — historical data preserved
  • UUID on rewards — safe for wallet transaction references

Considerations

  • Need new UserEvent type "mitra_approved" for mitra verification milestone
  • referral_codes.code needs unique index + collision handling
  • Anti-fraud: same device/IP detection deferred to Phase 2
  • Expiry: referrals can expire if referee doesn't complete milestones within campaign period
