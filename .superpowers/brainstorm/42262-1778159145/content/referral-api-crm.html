Referral Program — API & CRM Design

Backend endpoints + CRM page structure (scope: Backend + CRM only)

⚙️ Backend API Endpoints

Backoffice API — /api/v1/backoffice/referral-*
── Campaign Management ──
GET    /referral-campaigns?page=N&per_page=N&status=&target_role=
POST   /referral-campaigns
GET    /referral-campaigns/{id}
PUT    /referral-campaigns/{id}
DELETE  /referral-campaigns/{id}
PATCH  /referral-campaigns/{id}/status  ← {status: active|paused|ended}

── Tier Management (nested under campaign) ──
GET    /referral-campaigns/{id}/tiers
POST   /referral-campaigns/{id}/tiers
PUT    /referral-campaigns/{id}/tiers/{tierId}
DELETE  /referral-campaigns/{id}/tiers/{tierId}

── Milestone Management (nested under campaign) ──
GET    /referral-campaigns/{id}/milestones
POST   /referral-campaigns/{id}/milestones
PUT    /referral-campaigns/{id}/milestones/{milestoneId}
DELETE  /referral-campaigns/{id}/milestones/{milestoneId}

── Referral Records (read-only from CRM) ──
GET    /referrals?page=N&campaign_id=&status=&search=
GET    /referrals/{id}
PATCH  /referrals/{id}/flag  ← {reason: "suspected fraud"}

── Rewards (read-only + retry) ──
GET    /referral-rewards?page=N&referral_id=&status=
PATCH  /referral-rewards/{id}/retry  ← retry failed disbursement

── Analytics ──
GET    /referral-analytics/overview?campaign_id=&period=
GET    /referral-analytics/leaderboard?campaign_id=&limit=20
GET    /referral-analytics/tier-distribution?campaign_id=

⚡ Event Listener Flow

Automatic Milestone Processing
UserEvent created (e.g. "email_verified")
  ↓
ReferralMilestoneListener::handle($event)
  ↓ Check: does this user have a pending referral?
  ↓ Check: does the campaign have a milestone matching this event_type?
  ↓
ReferralMilestoneService::advanceMilestone($referral)
  ↓ Update referral.current_milestone_id
  ↓
ReferralRewardService::disburse($referral, $milestone)
  ├─ Calculate tier bonus for referrer
  ├─ Disburse referrer reward (cashback → WalletTransaction / voucher → VoucherUser)
  ├─ Disburse referee reward (if configured)
  └─ Update referral_codes.usage_count (on first milestone only)
  ↓
If all milestones completed → referral.status = "completed"

Job: ProcessReferralMilestone (queued, retryable)

🖥️ CRM Pages Structure

Sidebar: Marketing → Referral Program
📋 /dashboard/referral-campaigns — Campaign List
Table: name, target_role badge, status badge, period, total referrals, total rewards disbursed, actions (edit/toggle/delete)
Filters: status chips, target_role chips. Search by name.
📝 /dashboard/referral-campaigns/create — Create Campaign
Multi-section form: Basic Info → Milestones (repeater) → Tiers (repeater) → Review
Each milestone: event_type selector, referrer reward config, referee reward config
Each tier: name, min/max referrals, bonus %, extra perks
📊 /dashboard/referral-campaigns/[id] — Campaign Detail
Overview stats (total referrals, conversion rate, total disbursed) + milestone breakdown chart + tier distribution donut
Tabs: Overview | Referrals | Rewards | Leaderboard
👥 /dashboard/referrals — All Referrals
Table: referrer name, referee name, campaign, status badge, milestones progress, rewards given, created_at
Filters: campaign, status, date range. Search by referrer/referee name.
Actions: view detail, flag as fraud
🔍 /dashboard/referrals/[id] — Referral Detail
Referrer info, referee info, milestone timeline (visual progress), reward history, fraud flags

🔌 Integration Points (Backend)

Existing code modifications
1. RegisterService::handle()
  + Accept optional `referral_code` in registration data
  + After user creation: ReferralService::applyReferralCode($user, $code)
    → Creates referral record, triggers first milestone

2. EventTrackingService::trackEvent()
  + After tracking: dispatch ProcessReferralMilestone job
    → Checks if user is a referee with pending referral
    → Advances milestone if event matches

3. MitraMemberService (verification approval)
  + Track new event "mitra_approved" via EventTrackingService
    → Triggers milestone check for mitra referrals

4. New WalletTransaction type
  + Add "referral_reward" to transaction types
  + reference_type: "referral_reward", reference_id: reward UUID
